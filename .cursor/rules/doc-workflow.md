# ドキュメント参照ワークフロー

## 実装前に必ず読むファイル
以下のファイルを確認してから実装に着手すること。

1. `docs/02_management/PROGRESS.md` — 現在のフェーズとタスク状況を把握
2. `docs/01_concept/SPEC.md` — 要件・MVP スコープ・用語集を確認
3. `docs/01_concept/ARCH.md` — システム構成・DB/API 設計・ディレクトリ構成を確認

## 状況に応じて読むファイル
- UI に関わる実装 → `docs/01_concept/UI_UX.md`
- 技術選定の判断が必要 → `docs/01_concept/TECH_STACK.md`
- レビュー時 → `docs/03_review/REVIEW_CHECKLIST.md`
- タスク完了判断 → `docs/00_process/DEFINITION_OF_DONE.md`

## 実装後に更新するファイル
- `docs/02_management/PROGRESS.md` — 完了タスクのチェックボックスを更新
- `docs/02_management/LOGS/yyyy-mm-dd.md` — 作業内容を記録
- 設計変更があれば該当する `docs/01_concept/` のファイルも更新
