# DEFINITION_OF_DONE（完了の定義）

タスクや機能を「完了」と判断するための共通基準です。

## 機能実装の完了条件
- [ ] 要件（SPEC.md）に定義された仕様を満たしている
- [ ] エラーハンドリングが適切に実装されている
- [ ] ユニットテストが作成され、すべて通過している
- [ ] コードレビュー済み（REVIEW_CHECKLIST.md に基づく）
- [ ] 関連ドキュメントが更新されている（ARCH.md, SPEC.md 等）

## フェーズ完了の条件
- [ ] PROGRESS.md の該当フェーズのチェックボックスがすべて埋まっている
- [ ] 作業ログ（LOGS/）に実施内容が記録されている
- [ ] 未解決の課題がある場合、PROGRESS.md のメモ欄に記載されている

## リリースの完了条件
- [ ] 全テストが通過している
- [ ] 本番環境でのデプロイが成功している
- [ ] 動作確認（主要フローの手動テスト）が完了している
- [ ] 環境変数が正しく設定されている
- [ ] README にセットアップ手順が記載されている
