# WORKFLOW（開発手順）

各フェーズの入力・出力・使うツールを定義し、再現可能な開発フローを維持します。

## 開発フェーズ一覧

| # | フェーズ | 入力 | 出力 | 主担当 |
|---|---------|------|------|--------|
| 1 | 企画・要件定義 | アイデア・課題 | SPEC.md, UI_UX.md | Claude / 人 |
| 2 | 設計 | SPEC.md | ARCH.md, TECH_STACK.md, DB設計 | Gemini / 人 |
| 3 | 実装 | 設計ドキュメント一式 | ソースコード, テスト | Cursor / AI |
| 4 | 調整・品質向上 | 実装済みコード | リファクタ済みコード, テスト結果 | Cursor / 人 |
| 5 | 公開・運用 | 完成コード | デプロイ済み環境, CI/CD | GitHub Actions |

## フェーズ詳細

### フェーズ1: 企画・要件定義
- [ ] アイデアの言語化（SPEC.md の概要・ターゲットユーザー）
- [ ] MVP スコープの決定（SPEC.md の MVP スコープ）
- [ ] 画面遷移の整理（UI_UX.md）
- **完了条件**: SPEC.md と UI_UX.md が埋まっている

### フェーズ2: 設計
- [ ] 技術スタックの選定（TECH_STACK.md）
- [ ] システム構成の決定（ARCH.md）
- [ ] DB 設計（ARCH.md の DB設計セクション）
- [ ] API 設計（ARCH.md の API設計セクション）
- **完了条件**: ARCH.md が埋まっており、実装に着手できる状態

### フェーズ3: 実装
- [ ] 環境構築（README の手順で再現可能）
- [ ] 主要機能の実装
- [ ] テストの作成・実行
- **完了条件**: MVP 機能が動作し、テストが通る

### フェーズ4: 調整・品質向上
- [ ] UI/UX のリファイン
- [ ] コードレビュー（REVIEW_CHECKLIST.md）
- [ ] パフォーマンス確認
- **完了条件**: DEFINITION_OF_DONE.md の基準を満たす

### フェーズ5: 公開・運用
- [ ] インフラ構築・デプロイ
- [ ] CI/CD 設定
- [ ] ドキュメント最終確認
- **完了条件**: 本番環境でアクセス可能

## ブランチ運用
- `main`: 本番リリース用
- `develop`: 開発統合ブランチ
- `feature/xxx`: 機能開発ブランチ
- `fix/xxx`: バグ修正ブランチ
