# REVIEW_CHECKLIST（コードレビューチェックリスト）

PR / コードレビュー時に確認する共通チェックリストです。

## 基本
- [ ] 要件（SPEC.md）に沿った実装になっている
- [ ] 不要なコード・コメントアウトが残っていない
- [ ] 命名が直感的で一貫性がある（.cursorrules の規約に準拠）

## 品質
- [ ] エッジケースが考慮されている
- [ ] エラーハンドリングが適切
- [ ] テストが追加・更新されている
- [ ] パフォーマンスに問題がない（N+1クエリ、不要な再レンダリング等）

## セキュリティ
- [ ] ユーザー入力のバリデーションがある
- [ ] 秘匿情報（APIキー等）がハードコードされていない
- [ ] 認証・認可が適切に実装されている

## UI/UX（該当する場合）
- [ ] レスポンシブ対応
- [ ] ローディング・エラー状態の表示がある
- [ ] アクセシビリティに配慮されている
